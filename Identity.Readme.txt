Identity Breafing:

Now I see any wasm component must reside on client Project, may we start again from scratch with my Project? The goal is to develop an ASP.Net 10.0 Core api named Identity.Api that autenticates users for a pool of wasm apps which may share some but not all users. Same user may be enrolled on one or more apps and may have multiple roles. Users may login through either email and password or alternatively with external platforms like Apple or Google. Autentication through refresh tokens; persistance with EF Core. The template is the new unified Blazor Web App for .Net10.0 to be developed on a Snapdragon machine with VisualStudio 2026. The client Project will be instalable as pwa and Will take care of registering users, enrolling them on the different apps and assigning them the required roles on each app. 
Finally, a template for further solutions that will contain a .NET 10.0 asp.Net Core api [...].api with EF and a .NET 10.0 Wasm Project [...].Client Keep in mind that Blazored.SessionStorage is deprecated in .NET 10.0 No test projects in this moment. 
No Duende or dependencies that may involve pay licenses. 
The roadmap Will deliver instructions to paste on Powershell for:
- créate Identity Project
- nuget packages to add
- créate folders and empty files
- minimal code to launch the Project with no issues

# Root folder
mkdir IdentityPlatform
cd IdentityPlatform

# Create solution
dotnet new sln -n IdentityPlatform

# Create API (ASP.NET Core 10 Web API)
dotnet new webapi -n Identity.Api --use-controllers

# Create unified Blazor Web App (server + WASM interactivity)
dotnet new blazor -n Identity.Admin

# Add projects to solution
dotnet sln add Identity.Api/Identity.Api.csproj
dotnet sln add Identity.Admin/Identity.Admin.csproj

# Add the required nuget packages

cd Identity.Api

dotnet add package Microsoft.EntityFrameworkCore
dotnet add package Microsoft.EntityFrameworkCore.SqlServer
dotnet add package Microsoft.EntityFrameworkCore.Tools

dotnet add package Microsoft.AspNetCore.Authentication.JwtBearer
dotnet add package Microsoft.Extensions.Options.ConfigurationExtensions

cd ..

# Create folders + empty files

cd Identity.Api

mkdir Domain
mkdir Domain\Users
mkdir Application
mkdir Application\Auth
mkdir Infrastructure
mkdir Infrastructure\Persistence
mkdir Infrastructure\Auth
mkdir Controllers

ni Domain\Users\User.cs -ItemType File
ni Domain\Users\UserAppEnrollment.cs -ItemType File
ni Domain\Users\UserRole.cs -ItemType File

ni Infrastructure\Persistence\IdentityDbContext.cs -ItemType File
ni Infrastructure\Persistence\DesignTimeDbContextFactory.cs -ItemType File

ni Application\Auth\JwtOptions.cs -ItemType File
ni Application\Auth\RefreshToken.cs -ItemType File

ni Controllers\AuthController.cs -ItemType File

cd ..
